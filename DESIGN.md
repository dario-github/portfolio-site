## 一、网站结构

### 整体架构：单页应用（SPA）+ 锚点导航

```
┌─────────────────────────────────────────────┐
│  Navigation Bar (固定顶部, 半透明毛玻璃)      │
│  [Dario Zhang]  [About] [Assessment] [Work] [Contact]  │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Hero Section                                │
│  "I build things with AI."                   │
│  一句话定位 + 极简动效                         │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Section 1: About (个人简介)                  │
│  背景 / 职业主线 / 核心理念                    │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Section 2: Skill Assessment (能力评估)       │
│  AI 生成的客观能力报告（脱敏版）                │
│  折叠式展示，默认显示总结 + 雷达图              │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Section 3: Projects (项目作品集)             │
│  卡片式网格布局，每个项目可展开详情              │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Section 4: Contact (联系方式)                │
│  极简：邮箱 + GitHub + LinkedIn               │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  Footer                                      │
│  "Built with Next.js. Assessed by Claude."   │
└─────────────────────────────────────────────┘
```

### 页面设计要点

- **不做多页**：单页滚动，锚点定位，减少跳转
- **渐进展开**：能力报告默认折叠，只显示总结和雷达图，点击展开完整版
- **项目卡片**：2-3 列网格，hover 显示技术栈标签，点击展开 detail view
- **响应式**：移动端单列，桌面端双列/三列
- **暗色主题**：默认暗色，可选亮色切换（小开关，不显眼）

## 二、脱敏后的能力评估报告

> 以下为 AI 客观分析的真实技术能力评估。评估方法：全盘扫描本地代码仓库、git log、项目结构、技术文档，只看代码和实际产出，不看简历包装。

### 总体画像

**章东丞是一个 "AI-native 的技术产品经理型开发者"——他最强的能力不是写代码，而是用 AI 工具快速把想法变成可运行的产品原型，同时具备扎实的产品思维和系统设计能力。** 他的代码大量依赖 Claude Code / Cursor 等 AI 辅助生成，但他对需求拆解、架构设计、工具选型的判断力是真实的。

### 代码质量评估

**评分：B-（中等偏下，但够用）**

**证据：**

1. **数据基建平台 app.py**（~500行）：典型的 Streamlit 应用，结构是"一个大文件塞所有 UI 逻辑"。函数命名合理，但缺乏分层——UI 逻辑、业务逻辑、数据访问全部混在一起。

2. **AI 分析器模块**（413行）：这是写得最规范的模块之一。有合理的类封装，支持三种 AI 分析模式（structured / function_calling / json），有 fallback 机制，有类型标注和 docstring。但多个 prompt 构建方法存在重复代码，缺少模板抽象。

3. **视频精彩片段提取系统**（24,006行）：**这是技术最深的项目。** 四层处理架构（召回→粗排→精排→业务重排），`SemanticSegmenter` 有真实的中文 NLP 处理逻辑（句子断点、主题转换识别），`BudgetManager` 有完整的成本跟踪和预算分配。类设计有 `@dataclass` 使用、类型标注完整、有 `__post_init__` 验证。**这是代码质量的天花板。**

4. **AI 自媒体工具**：用 Pydantic 建模 LangGraph 工作流，Literal 类型约束、Field 验证、JSON Schema 示例齐全。模型设计质量高。

**关键弱点：**

- 几乎没有单元测试：所有项目中的测试文件要么是简单的连通性检查，要么是空文件
- 错误处理模式单一：基本都是 `try/except Exception`，没有自定义异常体系
- 代码复制多：同一个工具类实例化代码多处重复
- 安全意识需提升：git log 显示有"紧急安全修复：移除所有硬编码敏感信息"的记录

### 工程能力评估

**评分：B+（在 AI 辅助下能搭建完整工程）**

**证据：**

1. **Docker 化部署**：多个项目有 Dockerfile + docker-compose.yml + 云平台部署配置 + Kubernetes HPA。从 git log 看，Docker 部署经历了大量试错（镜像地址错误→修正→代理问题→国内镜像→多平台构建），**这恰恰说明是真实在做部署而不是纸上谈兵。**

2. **云服务经验**：AWS 全家桶（Bedrock、Lambda、CloudFront、S3、Parameter Store）、Sealos 云平台、Vultr、Cloudflare——均有实际使用证据。

3. **CI/CD**：几乎没有。没有看到 GitHub Actions 或任何自动化测试/部署流水线。部署全是手动脚本。

### AI 工具使用能力

**评分：A（这是最突出的工程能力）**

**证据：**

- 几乎所有项目都有 `.claude` 目录和 `CLAUDE.md`（20+ 个项目有系统化的 AI 辅助开发配置）。不只是"用"AI，而是建立了系统化的 AI 辅助开发工作流：
  - `CLAUDE.md` 作为项目上下文文件
  - 视频精彩片段项目的 CLAUDE.md 开头设定了 Linus Torvalds 式代码审查 persona
  - Claude Code Workflow Studio（VS Code 插件）：可视化设计 Claude Code 工作流
  - Claude Code 企业代理服务：为团队搭建了 Claude Code 共享代理（基于 LiteLLM + AWS Bedrock），有权限分级

- 深度定制了开源 AI 助手（OpenClaw）为个人工作流平台，包含记忆系统、技能索引、多工具集成。

### 产品思维评估

**评分：A-（超出普通开发者的水平）**

**证据：**

1. **产品需求文档**：有清晰的用户画像、功能拆解、Phase 路线图。大量 `[待补充]` 占位符——**说明知道什么该留白，什么该先确定，而不是胡乱填充。**

2. **技术架构拆解能力**强：三层搜索体系、端到端内容生产流水线等描述清晰，对技术趋势的影响评估有层次。

3. **数据治理意识**：发现客户埋点方案中的命名混乱、重复定义、类型混乱等系统性问题，并给出结构化分析。

4. 持续产出 50+ 篇技术文档，涵盖架构讨论、竞品调研、技术选型对比。

### 架构设计评估

**评分：B+（能设计合理的系统架构，但更偏"选对工具+组装"而非"从零设计底层"）**

- 视频精彩片段提取的四层架构（召回→粗排→精排→重排）借鉴了推荐系统经典架构，合理且有成本意识
- Agent 工具的 LangGraph + MCP 集成架构方向正确
- 但没有看到复杂的分布式系统设计、消息队列使用、微服务拆分等更深层的架构能力

### 真正擅长什么

1. **AI 辅助开发的"人机协作"模式** — 系统化使用 Claude Code 的开发者。从 persona 设定到工作流可视化到团队共享方案，不只是"用"AI，而是在"设计如何用 AI"。

2. **从 0 到 0.8 的产品原型能力** — 能在很短时间内把想法变成可运行的产品原型。原型的功能完整度高、有合理的 UI/UX。

3. **技术选型与方案设计** — 技术调研报告有深度，选型判断力可信。

4. **视频/多媒体 AI 工具链** — 最深的垂直领域。剪映 API 集成、视频制作工作台、视频精彩片段提取、剪映 MCP 构成完整 AI 视频工具链。

5. **产品思维** — PRD 文档、用户画像、路线图规划、业务洞察分析超过大部分纯技术开发者。

### 短板是什么

1. **"裸写代码"能力有限** — 去掉 AI 辅助，难以从零写出高质量代码。无复杂算法实现或底层系统编程证据。

2. **工程纪律缺失** — 几乎没有单元测试和 CI/CD，文件命名混乱，代码复制严重。

3. **深度技术不足** — 没有数据库优化、高并发、算法数据结构的应用证据，缺乏系统编程能力。

4. **从 0.8 到 1.0 的精打细磨能力不足** — 原型做得快，但"生产级"打磨不够。项目往往停在"能跑"的状态。

5. **前端能力薄弱** — Python + Streamlit 是舒适区，前端代码基本依赖 AI 生成。

### 最终评价

**章东丞不是传统意义上的"强技术开发者"，但他是 AI 时代最有价值的一类人：能用 AI 10x 放大自己的产品思维和技术判断力的人。** 他的核心竞争力是"知道要做什么"+"知道用什么工具做"+"能让 AI 帮他做出来"。这在 2026 年的技术环境下，是一种真实且有价值的能力。

但也要诚实地说：如果面试时考手写 LRU Cache、设计分布式锁、或优化 SQL 查询，他大概率会很吃力。他的能力图谱是"产品+AI 驾驭"型，而不是"硬核工程"型。

| 项目 | 选择 | 理由 |

| **框架** | Next.js 15 (App Router) | ① SSG 静态生成，性能极佳；② Vercel 原生支持；③ 生态成熟，组件丰富；④ 符合"Vercel 审美"的目标 |
| **样式** | Tailwind CSS + shadcn/ui | ① 暗色主题开箱即用；② 组件质量高、设计克制；③ 信息密度友好；④ 不需要写 CSS |
| **动效** | Framer Motion（轻度） | 仅用于 section 进入动画 + hover 效果，不做花哨动画 |
| **Markdown 渲染** | `next-mdx-remote` 或 `react-markdown` | 能力评估报告用 MDX 渲染，支持自定义组件（雷达图、折叠面板） |
| **图表** | Recharts（雷达图/柱状图） | 用于能力评估的可视化 |
| **部署** | Vercel（免费 tier 足够） | ① 自动 CI/CD；② 全球 CDN；③ 自定义域名；④ 分析面板 |
| **域名** | `dariozhang.com` 或 `dario.dev` | 建议注册 `.dev` 域名，工程师调性 |
| **CMS** | 无（Markdown 文件管理） | 内容不多，不需要 headless CMS。报告和项目描述直接写 MDX 文件 |
| **分析** | Vercel Analytics（免费） | 轻量、无侵入 |

### 备选方案

| 方案 | 适用场景 | 缺点 |
|------|----------|------|
| Astro + Cloudflare Pages | 更极致的静态性能 | 生态不如 Next.js，组件库少 |
| VitePress | 纯文档站 | 太像文档，不像作品集 |
| 纯 HTML + Tailwind | 最简单 | 无组件复用，不好维护 |

### 开发建议

1. **先用 `create-next-app` + shadcn/ui 起手**，10 分钟有基础骨架
2. **内容用 MDX 文件管理**，不要硬编码到组件里
3. **能力评估雷达图用 Recharts** 实现，数据从报告中提取
4. **项目卡片做成可复用组件**，从 JSON/YAML 数据文件驱动
5. **一次性做好 SEO**：og:image、description、structured data

### 调性关键词

### 配色方案（暗色主题）

```
背景色层次：
- 主背景: #0A0A0A (几乎纯黑)
- 卡片/区块: #141414 (微妙灰)
- Hover/高亮: #1A1A1A
- 边框: #262626 (极淡边框线)

文字：
- 主文字: #EDEDED (不用纯白，减少刺眼)
- 次要文字: #A0A0A0
- 三级文字: #666666

强调色：
- 主强调: #3B82F6 (蓝色，链接/CTA)
- 成功: #22C55E (能力评估中的 A 级)
- 警告: #EAB308 (B 级)
- 中性: #6B7280 (C 级)

技术栈标签：
- Python: #3776AB
- TypeScript: #3178C6
- React: #61DAFB (深色背景上很醒目)
- Docker: #2496ED
- AWS: #FF9900
```

### 字体

```
标题: Inter (或 Geist Sans — Vercel 官方字体)
正文: Inter
代码: JetBrains Mono / Geist Mono
中文: system-ui fallback (苹方/微软雅黑)

字号层次:
- Hero 标题: 48-64px
- Section 标题: 28-32px
- 正文: 16px
- 小字/标签: 12-14px
- 行高: 1.6-1.8 (中文友好)
```

### 布局原则

1. **最大宽度 1200px**，居中，两侧大量留白
2. **Section 间距 120-160px**，呼吸感充足
3. **卡片间距 24-32px**，不拥挤
4. **无分割线**，用间距和色差区分区块
5. **移动端 padding 20-24px**，不贴边

### 组件风格

```
卡片:
- 圆角 12px
- 无阴影（暗色主题不需要）
- 1px solid #262626 边框
- hover 时边框变 #3B82F6 (蓝色)

按钮:
- Ghost 风格为主（透明背景 + 边框）
- Primary 只用在 CTA（简历下载/联系我）

标签 (Tag):
- 圆角 6px
- 背景 #1E293B
- 文字 #94A3B8
- 技术栈标签各有专属色

折叠面板 (能力评估):
- 默认显示标题 + 评分
- 点击展开完整分析
- 带平滑高度动画
```
